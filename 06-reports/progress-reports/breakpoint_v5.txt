======================================================================
  THIRD: v5 IMPROVEMENTS (
======================================================================

  1. SMART DESCRIPTION ENGINE
     - Every property has a meaningful Arabic description
     - Examples: fill → "لون تعبئة الخلفية", draggable → "السماح بسحب العنصر"
     - Boolean properties include default value in description
     - Falls back to JSDoc if available, then smart generation

  2. COMPLETE CSS MAP (180+ mappings)
     - Every Konva property mapped to CSS equivalent where possible
     - Canvas-only properties (sceneFunc, hitFunc, etc.) marked as null
     - Typography, layout, visual, transform all fully mapped

  3. COMPONENT HIERARCHY
     - Each Radix component tracks its compound parts:
       dialog → [Root, Trigger, Portal, Overlay, Content, Title, Description, Close]
       select → [Root, Trigger, Content, Item, ItemText, ... etc]

  4. PROPER CATEGORIZATION
     - 9 categories: dimension, layout, visual, typography, transform,
       interaction, state, content, behavior, accessibility
     - Web props now have category field (missing in v4)

  5. EVENT DOCUMENTATION
     - All events have Arabic descriptions
     - Added mousewheel + contextmenu events to Konva shapes
     - Component-specific events (onEscapeKeyDown, onCloseAutoFocus, onSwipe)

  6. FIRST GOLDEN CLASSES
     - BaseComponent.js: 4-part architecture (CanvasRenderer, WebExporter,
       UI_Schema, LogicHandler). Supports children, events, serialization.
     - WebContainer.js: Layout container with flex, gap, padding, shadow.
       Renders via LeaferJS, exports to Tailwind HTML.
     - WebButton.js: Interactive button with variant/size/colorScheme.
       Full schema, canvas rendering, and Tailwind export.
